# 1ï¸âƒ£ Encapsulation

### Definition

Bundling data (variables) and methods (functions) together inside a unit (object/class), and restricting direct access to some parts.

### In JavaScript

```js
class BankAccount {
  #balance = 0;   // private field

  deposit(amount) {
    this.#balance += amount;
  }

  getBalance() {
    return this.#balance;
  }
}

const acc = new BankAccount();
acc.deposit(100);
console.log(acc.getBalance()); // 100
```

- `#balance` is private.
    
- You cannot access it directly.
    
- Data is protected inside the object.
    

Without classes (using closures):

```js
function BankAccount() {
  let balance = 0;

  return {
    deposit(amount) {
      balance += amount;
    },
    getBalance() {
      return balance;
    }
  };
}
```

Same idea.

### Definition

One object/class acquires properties and behavior from another.

In JavaScript (prototypal):

```js

  speak() {
    console.log("Animal speaks");
  }
}

  bark() {
    console.log("Woof");
  }
}

d.speak(); // inherited
d.bark();
```

Under the hood:

```
Dog.prototype â†’ Animal.prototype â†’ Object.prototype
```

This is delegation via prototype chain.

# 4ï¸âƒ£ Polymorphism

### Definition

Same method name, different behavior depending on object.

```js
class Shape {
  area() {}
}

class Circle extends Shape {
  area() {
    return 3.14 * 10 * 10;
  }
}

class Square extends Shape {
  area() {
    return 10 * 10;
  }
}
```

Both have `area()`, but behavior differs.

JS also supports dynamic polymorphism naturally:

```js
function makeSound(animal) {
  animal.speak();
}
```

As long as object has `speak()`, it works.

This is called **duck typing**.

# 6ï¸âƒ£ Association / Message Passing

```js
  }

    this.paymentService.pay();
  }
}
```

Objects interact via method calls.

# ðŸ”¥ Important: OOP in JavaScript is Different

JS is NOT classical OOP like Java or C++.

It is:

> Prototype-based OOP

Classes in JS are just syntax sugar over prototypes.

```js
class Hero {}
```

Becomes internally:

```js
function Hero() {}
Hero.prototype = { constructor: Hero }
```

So:

Inheritance in JS = prototype delegation.
